# altbox

![altbox logo](assets/img/altbox-box-logo.png)

Website for [altbox.dev](https://altbox.dev), the alternative toolbox for developers.

## About

altbox is a static site that showcases alternative CLI tools for Linux/macOS/Unix — modern
replacements for standard tools like `grep`, `ls`, and others. Built with [Astro](https://astro.build).

## Prerequisites

## Setup

``` bash
npm install
```

## Development

``` bash
npm run dev
```

Starts a local dev server at `http://localhost:4321` with hot reload.

## Building

``` bash
npm run build
```

Output goes to `dist/`. Deploy by rsyncing that directory to the server:

``` bash
make rsync
```

## Directory structure

``` text
content/tools/       Tool data files (one YAML file per tool)
public/
  tool/TOOLNAME/     Per-tool static assets (e.g. public/tool/ack/screenshot.png)
  assets/img/        Site-wide images (logo etc.)
src/
  layouts/           Base HTML layout (Layout.astro)
  lib/               Shared utilities (load tools, render Markdown)
  pages/             Astro pages — one file per URL pattern
    index.astro                  Home page (lists all tools)
    tool/[name].astro            Individual tool page
    alternative-to/[tool].astro  All tools alternative to a given standard tool
    language/[lang].astro        All tools written in a given language
    license/[license].astro      All tools with a given license
  plugins/           Custom remark plugin for screenshot shortcodes
  styles/            Global CSS
dist/                Built output (not in Git — generated by npm run build)
```

## Adding a tool

Create a new YAML file in `content/tools/`. The filename doesn't matter; the `name` field
is used for the URL slug. Example:

``` yaml
---
name: fd
author: David Peter
website: https://github.com/sharkdp/fd
project: https://github.com/sharkdp/fd
language: Rust
license: MIT
alternativeto: find
description: A simple, fast, and user-friendly alternative to find.
last_updated: 2026-03-15
screenshots:
    - file: fd/screenshot.png
      caption: fd finding files by name
body: |
    `fd` is a simple, fast alternative to `find`...

    {{screenshot: fd/screenshot.png, fd finding files by name}}
```

To list multiple alternatives, use a YAML list:

``` yaml
alternativeto:
    - du
    - df
```

The tool will automatically appear on:

- `/tool/fd`
- `/alternative-to/find`
- `/language/rust`
- `/license/mit`
- The home page
- The sidebar (for each `alternativeto` value that is new)

## Code style

- Use 4-space indentation in all files (JS, CSS, Astro, YAML)

## YAML conventions

- Start every file with `---`
- Do not include a `tags` field — it is unused
- `alternativeto` is a plain string for one value, a YAML list for multiple
- `alternativeto` values should be lowercase
- `last_updated` format is `YYYY-MM-DD`
- Leave `website:` blank (not omitted) if the tool has no separate website
- If `website` and `project` are the same URL, only `project` is needed — the tool page deduplicates them

## Screenshots

Put screenshot images in `public/TOOLNAME/`, e.g. `public/fd/screenshot.png`.
Reference them in the `body` field using:

``` markdown
{{screenshot: fd/screenshot.png, Caption text here}}
```

This is transformed into a `<figure>` element at build time by the remark plugin in
`src/plugins/remark-screenshots.js`.

## URL slugs

`language` and `license` values are lowercased for URLs. For example, `language: Rust`
produces the URL `/language/rust`, and `license: MIT` produces `/license/mit`.

`alternativeto` values are used as-is (they should already be lowercase, e.g. `grep`, `ls`).
A tool can have multiple `alternativeto` values — use a YAML list in that case.
